# Wireframe Review Feedback (Consolidated)

## 1. 목적

본 문서는 현재 wireframe이 기획 / 기능 / 흐름을 얼마나 정확히 반영하고 있는지에 대한 **통합 리뷰 결과**이며,
API/백엔드 구현 여부는 제외하고 **화면 구조와 워크플로우 표현에만 집중**한다.

---

## 2. 전체 평가 (Overall Verdict)

현재 wireframe은 주요 도메인 구조와 단계(step2~step6)를 **형식적으로는 포함**하고 있으나,
실제 사용자 입장에서 **업무 흐름이 실행 가능한 형태로 표현되지 않았다**.

핵심 문제는 다음과 같다:

* 흐름이 화면에서 “보이지 않음”
* 단계 간 연결이 끊겨 있음
* 분기/루프/결정 구조가 UI에 표현되지 않음

👉 결론: **Revise (구조 수정 필요)**

---

## 3. 잘 반영된 부분 (Strengths)

* Step2 ~ Step6까지의 큰 단계 구조 존재
* 사모사채 도메인 특화 분석 요소 (후순위채 / 영구채 등) 반영
* Step3 분기 구조 자체는 존재
* Step4에서 종합 분석 영역 존재
* 보고서/조건/사후관리 개념 포함

---

## 4. 핵심 문제 요약 (Critical Issues)

### 4.1 Step5 / Step6 붕괴

* 보고서 작성 / 제출 / 승인 / 결과 기록이 한 화면에 혼합
* 단계 간 책임과 흐름이 명확하지 않음

👉 해결 필요: **Step5 / Step6 완전 분리**

---

### 4.2 분기 구조 무력화

* Step2에서 분기 선택 가능
* 하지만 Step3/이후 흐름에 영향 없음

👉 문제:

* “선택했지만 시스템이 반영하지 않음”

👉 해결:

* 분기 상태 유지 + Step4에서 merge 구조 명확화

---

### 4.3 보완 요청 루프 부재 (가장 중요한 문제)

현재:

* 누락 체크 있음
* 하지만 이후 행동 없음

필요한 구조:

```text
누락 발견
→ 보완 요청
→ 제출자 응답 대기
→ 재업로드
→ 재검증
→ 메인 흐름 복귀
```

👉 이 루프가 UI에 없음 = **핵심 업무 흐름 손실**

---

### 4.4 Human 판단 위치 오류

현재:

* Step3에서 판단/승인/서명 등장

정상:

* Step4에서 판단 (AI 결과 이후)

👉 문제:

* 분석 단계와 판단 단계가 혼합됨
* 책임 구조 붕괴

---

### 4.5 “화면은 있는데 행동이 없음”

현재:

* 카드 / 데이터 / 결과 있음
* 하지만 사용자 행동 정의 없음

부족한 요소:

* 무엇을 검토해야 하는지
* 무엇을 결정해야 하는지
* 다음 단계 조건이 무엇인지

👉 결과:

* UI는 있지만 “업무 수행 불가능”

---

## 5. 누락된 구조 요소 (Missing Structural Elements)

### Step2

* 비교 / 과거 사례 / 내부 기준 확인 (2.5)
* 보완 요청 UI
* 재업로드 흐름

### Step3

* 분석 결과 검토 행동
* branch 결정 유지
* 진행 조건 판단

### Step4

* Human confirm (확정 / 수정 / 거절)
* 메모 / 보완 요청 / 조건 변경
* 판단 기록

### Step5

* 제출 준비 상태
* unresolved 항목 체크

### Step6

* 승인 / 반려 / 재작업
* 결과 기록 / 통보

---

## 6. 흐름 문제 (Broken Flow)

* Step2 → Step3 분기 연결 깨짐
* Step3에서 분기 재선택 발생
* Step4가 “결정 단계”가 아님
* Step5/6가 하나로 합쳐짐
* rejection / rework 흐름 없음
